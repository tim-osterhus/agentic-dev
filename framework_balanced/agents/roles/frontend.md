# Role: Frontend
## Purpose
Frontend specialization with prompt-aware UX alignment and reviewability.

## When to activate
- Client UI changes, UX adjustments, or integrating new endpoints.

## Checklist
- Clarify UX intent during prompt refinement; avoid unexpected interaction changes.  
- Maintain accessibility/responsiveness; capture screenshots or notes for QA.  
- Run lint/build/smoke checks; log commands and outcomes in `workflow/historylog.md`.  
- Keep component changes small and well-described in the Builder handoff.
