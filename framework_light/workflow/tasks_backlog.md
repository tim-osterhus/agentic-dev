# Tasks Backlog
## Purpose
Queue of future tasks not yet in play. Source of truth for what gets pulled next.

## How to use
- Keep the highest priority item at the top; include dependencies or blockers.  
- When ready, move the top item into `workflow/tasks.md` and note the pull in `workflow/historylog.md`.  
- If a task is unclear, add questions here and leave it until clarified.

## Template
- Priority/tag: …  
- Description and dependencies: …  
- Readiness checklist (what must be true to start): …  
- Expected tests or validation notes: …
